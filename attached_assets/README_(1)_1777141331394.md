# OnePort 365 — Replit Setup Guide

## What's in this folder

| File | What it is |
|------|-----------|
| `outlook_scan.html` | RFQ intake UI — inbox, email reader, extraction panel, RFQ tracker |
| `rfq_pipeline.html` | RFQ pipeline tool — 4-step workflow, PDF upload, quote generation, RFQ dashboard |
| `README.md` | This file |

---

## Step 1 — Create a new Replit

1. Go to [replit.com](https://replit.com) and sign in
2. Click **+ Create Repl**
3. Choose template: **HTML, CSS, JS**
4. Name it: `oneport365`
5. Click **Create Repl**

---

## Step 2 — Upload the files

1. In the Replit file panel (left sidebar), click the **three dots** next to Files
2. Select **Upload file**
3. Upload both files:
   - `outlook_scan.html`
   - `rfq_pipeline.html`
4. Delete the default `index.html` that Replit creates — or rename `outlook_scan.html` to `index.html` to make it the home page

---

## Step 3 — Add your Anthropic API key

Both files need your API key to run the AI extraction and generation steps.

Open each file and find this line near the top of the `<script>` block:

```javascript
const ANTHROPIC_API_KEY = 'YOUR_API_KEY_HERE';
```

Replace `YOUR_API_KEY_HERE` with your actual key from:
**https://console.anthropic.com → API Keys → Create Key**

Do this in **both files**.

> ⚠️ Keep this key private. Do not share the files publicly with the key inside.

---

## Step 4 — Run

1. Click the green **Run** button at the top of Replit
2. A preview panel will open on the right
3. To switch between the two tools, click the file name in the top bar of the preview, or navigate directly:
   - `https://your-repl-name.repl.co/outlook_scan.html`
   - `https://your-repl-name.repl.co/rfq_pipeline.html`

---

## Step 5 — Allow API calls (important)

Replit runs files in a browser context. The Anthropic API calls will work **only if** the `anthropic-dangerous-direct-browser-access: true` header is present in every fetch call — this is already included in both files. No extra setup needed.

If you see a **CORS error** in the browser console:
- Open the file
- Find the `HEADERS` constant
- Confirm it includes: `'anthropic-dangerous-direct-browser-access': 'true'`

## How to use each tool

### outlook_scan.html — RFQ Intake
| Tab | What it does |
|-----|-------------|
| New RFQs | Shows incoming emails from the commercial inbox. Click any email to read it and see extracted fields in the right panel. |
| RFQ Tracker | Full table of all RFQs with status — Info needed, Ready, Rate requested, Quote sent. Click any row for full details. |

### rfq_pipeline.html — RFQ Pipeline
| Step | What it does |
|------|-------------|
| 1 — Read RFQ | Paste email text or upload PDF. Extracts all shipment fields. |
| 2 — Rate Request | Generates a vendor rate request email with no customer name. |
| 3 — Vendor Response | Paste vendor reply or upload rate PDF. Parses all rates. |
| 4 — Generate Quote | Adjust margin, generate formatted customer quote. |
| RFQs button | Opens the RFQ dashboard — save, resume, and track sessions by RFQ number. |

---

## Minimum fields required to get rates

For an RFQ to move from **Info needed** to **Ready**, the following must be present:

1. Port of loading (POL)
2. Port of discharge (POD)
3. Commodity
4. Container type (20FT / 40FT / 40HC)
5. Container quantity
6. Cargo class (GC or DG)
7. Incoterm

If any of these are missing, a follow-up email is drafted automatically and sent to the customer after 5 minutes.
